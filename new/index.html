1. Problem

SMEs need early liquidity warnings, but pure LLM financial analysis can hallucinate math and bypass controls.

2. Architecture

FastAPI validates events, the server binds trusted context, and every tool call passes through policy.

3. Engines

Data, risk, peer benchmark, scenario, and intervention engines own calculations and provenance.

4. Evidence

Tests, immutable database checks, prompt-injection blocks, PII redaction, and trace audits support the claim.

Main claim: the LLM is not trusted with financial computation or database mutation. The model-safe layer can only explain deterministic outputs and request draft-only actions.
