KAGGLE / NOTEBOOK STORY SECTIONS
--------------------------------

[ Title ]
  CashFlow Guardian: Secure Policy-Gated SME Liquidity Intelligence

[ Section 1 - Problem ]
  SME liquidity crises require early warnings, but pure LLM math is unsafe.

[ Section 2 - Architecture ]
  FastAPI validates events, server binds SecurityContext, policy gates every tool.

[ Section 3 - Deterministic Engines ]
  Data, risk, benchmark, scenario, and intervention engines own the calculations.

[ Section 4 - Agent Role ]
  Google ADK explains and routes outputs; it cannot mutate the source database.

[ Section 5 - Security Evidence ]
  Prompt injection blocking, PII redaction, RBAC denial, HITL approval boundary.

[ Section 6 - Evaluation Evidence ]
  Passing tests, immutable DuckDB checks, audit traces, smoke trace validation.
